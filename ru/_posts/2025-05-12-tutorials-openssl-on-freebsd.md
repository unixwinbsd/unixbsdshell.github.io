Одной из программ, требующих особого внимания со стороны пользователей FreeBSD, является набор программ OpenSSL, включенный по умолчанию в систему FreeBSD. OpenSSL предоставляет уровень шифрования поверх уровня обычного подключения; поэтому он может использоваться многими сетевыми приложениями и службами.

OpenSSL можно использовать для шифрования подключений к почтовым клиентам, интернет-транзакций, таких как кредитные карты, и многого другого. Многие порты, такие как www/apache13-ssl и mail / sylpheed-claws, созданы с использованием OpenSSL.

OpenSSL  это инструментарий с открытым исходным кодом и криптографическая библиотека, реализующая протоколы SSL (Secure Sockets Layer) и TLS (Transport Layer Security).

OpenSSL  это независимый проект, управляемый добровольцами со всего мира. Короче говоря, OpenSSL предоставляет криптографическое устройство для защиты сетевых подключений. Распространенной реализацией SSL является защита веб-страниц с помощью HTTPS (протокол передачи гипертекста, защищенный шифрованием). Шифрование OpenSSL по протоколу HTTP состоит из следующих этапов:

1. HTTP-клиент (веб-браузер) отправляет HTTPS-запрос на веб-сервер.
2. В ответ сервер отправляет клиенту SSL-сертификат, содержащий открытый ключ, доменное имя и выданный центр сертификации SSL.
3. Клиенты отправляют сообщения, зашифрованные с использованием открытого SSL-ключа сервера.
4. Сервер расшифровывает это сообщение с помощью своего закрытого SSL-ключа.
5. В конечном итоге сервер отправляет клиенту расшифрованное сообщение.
6. Если клиент получает правильное сообщение, обе стороны могут начать безопасный обмен информацией при условии, что клиент доверяет центру выдачи сертификатов.

OpenSSL предоставляет необходимые инструменты для создания запросов на подписание сертификатов, закрытых серверных ключей и самозаверяющих сертификатов. В сочетании с признанным центром сертификации можно создать сертификат доверенного сервера для использования с протоколами TCP, такими как HTTP, SMTP, IMAP и т.д.

OpenSSL основан на библиотеке SSLeay, первоначально разработанной Эриком А. Янгом и Тимом Дж. Хадсоном. SSLeay - это реализация протокола Secure Socket Layer от Netscape с открытым исходным кодом, который использовался в браузерах Netscape Secure Server и Navigator в середине 1990-х годов.

Версия OpenSSL, входящая в состав FreeBSD, поддерживает протоколы сетевой безопасности Secure Sockets Layer v2/v3 (SSLv2/SSLv3) и Transport Layer Security v1 (TLSv1) и может использоваться в качестве основной криптографической библиотеки.

## 1. Обновите OpenSSL
Чаще всего OpenSSL используется для создания сертификатов, используемых пакетами программного обеспечения. Этот сертификат подтверждает, что данные компании или физического лица указаны правильно и не изменены. Если соответствующий сертификат не был проверен одним из нескольких центров сертификации (Cas), обычно выводится предупреждение.

Центр сертификации - это компания, такая как VeriSign, которая подписывает сертификат для проверки личности физического лица или компании. Эта процедура не является бесплатной и не является абсолютно необходимой для использования сертификата; однако она может убедить некоторых очень осторожных пользователей.

В последних версиях FreeBSD вам не нужно устанавливать OpenSSL, поскольку OpenSSL автоматически устанавливается в системах FreeBSD. Несмотря на это, иногда встроенная в систему версия не является последней. Нет ничего плохого в том, что вы обновите его, переустановив OpenSSL. Процесс установки OpenSSL очень прост. Вот как установить OpenSSL в систему FreeBSD.

```
root@ns4:~ # cd /usr/ports/security/openssl
root@ns4:/usr/ports/security/openssl # make DISABLE_VULNERABILITIES=yes install clean
root@ns4:/usr/ports/security/openssl # echo "DEFAULT_VERSIONS+=ssl=openssl" >> /etc/make.conf
```
В процессе установки появится меню выбора зависимостей. Вы можете включить или отключить показанные опции, как показано на следующем рисунке.

![OpenSSL FreeBSD Install](https://gitflic.ru/project/iwanse1212/basics-of-using-openssl-on-freebsd-14/blob/raw?file=Openssl%20freebsd%20tutorials.jpg&commit=bbb6e4fed11a184a19362fa2670cb53f40bf954e)

После этого приступайте к процессу настройки. Перед настройкой OpenSSL создайте резервную копию или скопируйте файлы openssl.cnf, чтобы в случае возникновения ошибки во время настройки исходный файл оставался на месте.
