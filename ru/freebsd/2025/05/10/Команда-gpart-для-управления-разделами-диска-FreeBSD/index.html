    Команда gpart для управления разделами диска в FreeBSD 1

    FreeBSD продолжает обновлять свою систему, как будто остановки не видно. Некоторые командные строки, которые больше не подходят для текущих условий, заменяются более новыми, лучшими командами. Как и команды bsdlabel и fdisk, они больше не могут использоваться. Даже разработчики FreeBSD забыли об этих двух командах.

    Вместо этого FreeBSD устанавливает команду gpart в качестве команды по умолчанию для управления и организации дисков. Эта команда удивительно способна управлять разделами на новейших жестких дисках, размеры которых достигают терабайт. На самом деле gpart способен очень эффективно и быстро создавать, изменять и удалять разделы.

    В этой статье представлено полное руководство по использованию команды gpart для управления разделами во FreeBSD, в котором рассматриваются ее функции, команды и практические примеры.

    А. О команде gpart

    Прежде чем изучать gpart, неплохо было бы узнать, что такое gpart. gpart GEOM Partitioning — это встроенная командная строка оболочки FreeBSD для управления таблицами разделов диска или их восстановления, особенно если первичная таблица разделов была повреждена или утеряна. Команда gpart работает путем считывания диска сектор за сектором и использования различных модулей распознавания файловой системы для определения потенциальных границ и типов разделов.

    Эта утилита поддерживает различные схемы разделов, включая GPT (таблица разделов GUID), MBR (главная загрузочная запись) и метки BSD. По сути, gpart помогает восстановить утерянную или поврежденную информацию о разделах, тем самым позволяя пользователям получить доступ к данным или заново создать разделы.

    gpart имеет основные функции, которыми вы можете воспользоваться, в том числе:

    1. Поддержка нескольких схем разбиения на разделы: gpart поддерживает метки GPT, MBR и BSD, что делает его универсальным для различных вариантов использования.
    2. Управление разделами: легкое создание, удаление, изменение размера и модификация разделов.
    3. Управление атрибутами: установка и изменение атрибутов раздела, таких как флаг загрузки и тип раздела.
    4. Управление загрузочным кодом: установка и управление загрузочным кодом для различных архитектур.
    5. Возможности скриптов: команды gpart можно записывать в скрипты, что делает их пригодными для автоматизированных задач.

    В своем использовании gpart очень гибок и позволяет пользователям создавать, изменять и удалять разделы, а также очень легко управлять атрибутами разделов и загрузочным кодом. gpart способен сканировать диск сектор за сектором, ища шаблоны и сигнатуры, указывающие на начало различных файловых систем или типов разделов, поскольку gpart специально разработан для восстановления информации о разделах, когда первичная таблица разделов (расположенная в секторе 0 диска) повреждена или утеряна.

    B. Основные команды gpart (флаги)

    У gpart есть много опций, которые вы можете использовать. Все эти опции необходимо использовать по назначению, в соответствии с их функцией. Каждая опция gpart отличается от другой. Есть даже варианты, которые можно комбинировать.

    Полный список команд gpart вместе с их параметрами можно просмотреть, введя команду gpart -h, как в следующем примере.

    C. Как использовать команду gpart

    После того, как вы узнаете, что такое gpart, его функции и различные возможности, в этом разделе мы приведем примеры использования каждой команды, имеющейся в gpart. В этой статье мы используем носитель Flashdisk для практики использования gpart. Однако каждую из предоставленных нами команд можно также запустить на жестком диске, вам нужно только изменить имя диска, например, da0 для флэш-диска, если вы используете жесткий диск, замените da0 на ada0.

    Прежде чем использовать команду gpart, проверьте, сколько дисков на вашем сервере FreeBSD. Выполните следующую команду, чтобы узнать количество дисков в системе FreeBSD.

<JetFlash Transcend 16GB 1100>     at scbus3 target 0 lun 0 (da0,pass1)

    На вашем компьютере установлено только 2 диска.

    1. Жесткий диск марки WDC (western digital) с маркировкой ada0, и
    2. Флэш-диск марки Transcend с меткой da0

    В этой статье мы используем da0 только в качестве практики. Если вы хотите узнать права собственности и разрешения для файлов на Flashdisk da0, используйте команду ps.

    root@ns4:~ # ls -l /dev/da0

    а.3. gpart показать параметры

    Команда gpart show имеет несколько параметров, включая:

    Команда gpart show имеет несколько параметров, включая:

    1. -l : Для схем разделов, поддерживающих метки разделов, печатать метки вместо типа раздела.
    2. -p : Отображает имя поставщика вместо индекса раздела.
    3. -r : отображает необработанные типы разделов вместо символических имен.

    Пример использования этой опции показан ниже.

    root@ns4:~ # gpart show -l
root@ns4:~ # gpart show -l da0
root@ns4:~ # gpart show -p ada0
root@ns4:~ # gpart show -r da0

    б. Создание схемы разделов - gpart create

    Команда gpart create используется для создания новой схемы разделов на флэш-диске или жестком диске. Используемую схему необходимо указать с помощью параметра -s scheme. Поэтому, прежде чем запустить команду gpart add. Сначала необходимо выполнить команду gpart create (не отменяйте команду).

    Например, вы хотите создать новый раздел, а именно GPT на Flashdisk da0. Команда gpart create инициализирует da0 со схемой разделов GPT.

    root@ns4:~ # gpart create -s gpt /dev/da0
da0 created

    При желании вы можете заменить GPT на MBR или BSD, если вам больше нравятся эти схемы.

    root@ns4:~ # gpart create -s bsd da0
root@ns4:~ # gpart create -s mbr da0

    После завершения выполнения указанной выше команды продолжите выполнение команды gpart add.

    в. Добавление нового раздела - gpart add

    После создания схемы разделов вы можете добавлять новые разделы на Flashdisk. Тип раздела необходимо указать с помощью параметра -t. Расположение, размер и другие атрибуты раздела будут рассчитаны автоматически, если соответствующие параметры не указаны.

    Приведенный ниже пример команды добавит раздел размером 10 ГБ на Flashdisk (da0). Обратите внимание на написание команды.

    root@ns4:~ # gpart add -t freebsd-ufs -s 10G da0
da0p1 added

    Давайте посмотрим на результаты команды выше.

    root@ns4:~ # gpart show da0
=>      40  30834608  da0  GPT  (15G)
        40  20971520    1  freebsd-ufs  (10G)
  20971560   9863088       - free -  (4.7G)

    В приведенной выше команде мы видим, что раздел размером 10 ГБ с типом freebsd-ufs (используется для файловой системы UFS) был добавлен в da0. Вы можете указать различные типы разделов, например freebsd-zfs для ZFS или linux-data для разделов Linux.

    Если вы работаете с жестким диском, который больше флэш-диска. Можно ли создать несколько разделов с помощью gpart? У gpart много преимуществ: он может создавать много разделов на одном диске.

    В приведенном ниже примере мы создадим два раздела на флэш-диске объемом 16 ГБ. Первый раздел мы создадим без указания начального LBA, но размером 1 ГБ (2097152 блока). На втором разделе без указания начального LBA и без указания его размера. Таким образом, разделы будут созданы на всем свободном пространстве.

    root@ns4:~ # gpart add -t freebsd-swap -s 2097152 /dev/da0
da0p1 added
root@ns4:~ # gpart add -t freebsd-ufs /dev/da0
da0p2 added
root@ns4:~ # gpart show da0
=>      40  30834608  da0  GPT  (15G)
        40   2097152    1  freebsd-swap  (1.0G)
   2097192  28737456    2  freebsd-ufs  (14G)

root@ns4:~ #

    Объяснение приведенной выше команды следующее:

    Первая команда используется для создания файла подкачки размером 1 ГБ (2097152 блока).

    Вторая команда используется для создания файловой системы UFS, при этом оставшийся размер занимает файл подкачки.

    Третья команда просматривает разделы.

    В таблице ниже показаны параметры команды gpart add, которые вы можете использовать:

    Option Description
    -a alignment If specified, the gpart utility attempts to align the starting offset and partition size to be multiples of the alignment values.
    -b start The logical block address where the partition will start. SI unit suffixes are allowed.
    -f flags Additional operational options
    -i index The index in the partition table where the new partition will be placed. The index specifies the device-specific filename used to represent the partition.
    -l label The label attached to the partition. This option is only valid when used on a partition scheme that supports partition labels.
    -s size Create a partition with size size. SI unit suffixes are allowed.
    -t type Create a partition with type type. Partition types are discussed below in the section titled “PARTITION TYPES”.

    Вы можете объединить эти параметры в одну команду.

    root@ns4:~ # gpart add -b 1058 -s 2G -l namalabel -a 1M -t freebsd-ufs da0
da0p1 added

    Чтобы увидеть имя метки раздела, который вы создали выше, выполните следующую команду.

    root@ns4:~ # diskinfo /dev/gpt/namalabel
/dev/gpt/namalabel      512     2147483648      4194304 0       1048576 261     255     63

    г. Удаление схем и разделов - gpart destroy

    Вам следует обратить внимание на то, что gpart destroy — это не то же самое, что gpart delete. Команда gpart destroy используется для удаления схемы разделов и всех разделов диска. Это означает, что все команды, которые вы запускаете с помощью gpart create и gpart add, будут удалены с помощью gpart destroy.

    Например, в предыдущем разделе вы узнали, как создать схему разделов GPT и создать раздел файловой системы UFS на Flashdisk (ad0). С помощью gpart destroy все, что вы создадите, будет уничтожено. Как запустить команду gpart для уничтожения схемы и разделов в ad0, можно увидеть ниже.

    root@ns4:~ # gpart уничтожить -F da0

или

root@ns4:~ # gpart уничтожить -F /dev/da0

    Приведенная выше команда удаляет все разделы и схемы gpt из da0. Опция -F используется для принудительного уничтожения таблицы разделов, даже если она не пуста. Используйте эту команду с осторожностью, так как она сотрет все данные на диске.

    е. Удаление раздела - gpart delete

    В отличие от команды gpart destroy, команда gpart delete удаляет только существующие разделы на диске. Это означает, что gpart delete удалит только ту команду, которую вы запустили с помощью gpart add. При этом схема разделов, созданная вами с помощью gpart create, не удаляется. Итак, разница между gpart destroy и gpart delete совершенно очевидна.

    Хорошо, чтобы вы не запутались, вот различия между gpart destroy и gpart delete.

    1. gpart destroy: удаляет результаты команд gpart create и gpart add (схемы или таблицы и разделы).
    2. gpart delete: удаляет только результаты команды gpart add (раздел диска).

    Команда gpart delete имеет опцию -i (индекс), эту опцию всегда следует использовать при удалении раздела. На изображении ниже показан пример флэш-диска с 6 индексами разделов.

    Обратите внимание на столбец da0, числа от 1 до 6 — это индекс раздела. Этот индекс очень важен для удаления разделов. Ниже приведен пример использования команды gpart delete с индексом, как показано на рисунке выше.

    root@ns4:~ # gpart delete -i 3 da0
da0p3 deleted

    Приведенный выше пример команды удаляет третий раздел индекса. Это означает, что раздел с порядковым номером 3 был удален. Чтобы убедиться в этом, выполните команду gpart show da0.

    ф. Подтверждение и отмена изменений раздела — gpart commit undo

    Полезной функцией утилиты gpart является распознавание «отложенных» изменений. Это означает, что если вы добавите опцию -f x, то изменения не будут записаны на диск немедленно, а только после того, как вы выполните команду gpart commit.

    По умолчанию системы FreeBSD запускают команду gpart с параметром «C», что означает, что каждая запущенная вами команда gpart немедленно фиксируется или выполняется. Чтобы предотвратить немедленное выполнение каждой команды gpart, можно указать параметр «-f x», который заставит каждую команду gpart вносить ожидающие изменения. Вы можете немедленно зафиксировать эти отложенные изменения или отменить их с помощью команды gpart undo.

    ф.1. команда отмены gpart

    Поэтому, как правило, команды gpart commit и gpart undo всегда используются парами. Чтобы было понятнее, приведем примеры команд gpart commit и gpar undo.

    root@ns4:~ # gpart create -f x -s gpt da0

    В приведенной выше команде создайте схему разделов на Flashdisk (da0). Обратите внимание на опцию -f x. С помощью этой опции вы пока не можете создать схему разделов, поскольку ее необходимо сначала зафиксировать или отменить.

    Если вы хотите отменить указанную выше команду, используйте gpart undo. Прежде чем выполнить команду отмены. Просмотр разделов на Flashdisk.

    root@ns4:~ # gpart show da0
=>      40  30834608  da0  GPT  (15G)
        40  30834608       - free -  (15G)

    После этого выполните команду отмены gpart.

    root@ns4:~ # gpart undo da0

    Запустите команду gpart show еще раз и посмотрите результаты. Команда gpart create для создания новой схемы разделов отменена.

    ф.2. команда gpart commit

    Поскольку нашей целью является создание новой схемы разделов, команду gpart create можно выполнить напрямую. Рассмотрим пример команды ниже, чтобы создать новую схему разделов с опцией -f x и немедленно ее зафиксировать.

    root@ns4:~ # gpart create -f x -s gpt da0

    Поскольку приведенная выше команда использует опцию -f x, чтобы применить ее, просто выполните команду gpart commit.

    root@ns4:~ # gpart show da0

    Если команда gpart create или любая другая команда gpart была зафиксирована, то команда gpart undo не может быть использована. В качестве доказательства запустите gpart undo.

    root@ns4:~ # gpart undo da0
gpart: Operation not permitted

    Появляется сообщение «Операция не разрешена», поскольку указанная выше команда gpart create была применена или зафиксирована с помощью gpart commit.

    Потому что обсуждение команды gpart довольно длинное. В этой статье мы разделим ее на две части. В этой первой статье мы закончим командами gpart commit и undo. Во второй статье мы продолжим рассмотрение других команд gpart.
