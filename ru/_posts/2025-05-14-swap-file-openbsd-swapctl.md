Когда вы используете OpenBSD, система использует сразу два типа памяти, а именно оперативную память (RAM) и файлы подкачки. В отличие от оперативной памяти, в которой данные сохраняются автоматически, файл подкачки будет работать, если он настроен на жестком диске. Другое отличие заключается в том, что в оперативной памяти хранятся только временные данные. При выключении компьютера из оперативной памяти будут удалены все хранящиеся в ней данные. В то время как файлы подкачки могут постоянно хранить данные на жестком диске.

OpenBSD, как и другие операционные системы, использует концепцию, называемую виртуальной памятью. Эта концепция используется для предотвращения нехватки памяти из-за высокой нагрузки на запущенные прикладные программы. Это достигается путем добавления пространства подкачки в систему OpenBSD. Пространство подкачки используется для хранения тех страниц памяти, которые не помещаются в физическую память.

Все данные, которые не хранятся в физической памяти, будут сохранены на жестком диске. Пространство подкачки, которое существует на этом жестком диске, называется файлом подкачки. Другими словами, файл подкачки - это резервный жесткий диск, когда физическая память заполнена или перегружена. Таким образом, производительность системы не будет нарушена.

## 1. Понимание работы с Файлом подкачки

Файл подкачки - это также файловая система, предназначенная для создания места для хранения на жестком диске или твердотельном накопителе (SSD). Обычно это происходит, когда заканчивается оперативная память компьютера. Чтобы освободить место для других программ, файл будет заменен на часть оперативной памяти запущенной программы. Обратите внимание на приведенное ниже изображение процесса замены в работающей операционной системе.

gambar

Файл подкачки больше похож на виртуальную память, чем на физическую оперативную память. Если у вас есть файл подкачки, ваша система будет делать вид, что у нее больше оперативной памяти, чем у вас есть. Программы, используемые в оперативной памяти, и список файлов будут преобразованы на жесткий диск до тех пор, пока они не понадобятся. Это позволяет увеличить объем оперативной памяти с помощью жесткого диска.

Файл подкачки - это название файла, хранящегося на жестком диске, из которого операционная система извлекает данные из оперативной памяти. Файл подкачки служит для имитации объема оперативной памяти, превышающего объем физической оперативной памяти, имеющейся в компьютере.

Система обмена файлами позволяет компьютерам с небольшим объемом оперативной памяти запускать большие и сложные программы. Когда в оперативной памяти компьютера заканчивается свободное место, операционная система может перенести часть данных, хранящихся в оперативной памяти, в специальный файл на жестком диске, известный как файл подкачки. Это освобождает место в оперативной памяти и позволяет компьютеру продолжать работать.

Размер файла подкачки может варьироваться в зависимости от операционной системы и объема физической оперативной памяти вашего компьютера. Однако важно убедиться, что файл подкачки находится на жестком диске с достаточным количеством свободного места, чтобы система могла эффективно обмениваться данными.

## 2. Использование команд swapctl и swapon

Команда swap в OpenBSD почти такая же, как и в NetBSD, в OpenBSD программа swapctl используется для добавления, удаления, составления списка и определения приоритетов устройств и файлов подкачки в системе. принимая во внимание, что программа swapon действует так же, как swapctl-a, за исключением того, что сама swapon вызывается с помощью-A, и в этом случае она действует как swapctl-A.

Чтобы выполнить команду swapctl, вы должны понимать некоторые из прилагаемых к ней опций, как показано в следующей таблице.

| ----------- | -----------   | 
| -A          | This option causes swapctl to read the /etc/fstab file for devices and files with an “sw” type, and adds  
all these entries as swap devices. If no swap devices are configured, swapctl will exit with an error code.          |

### 2.1. Создание файла подкачки

Чтобы добавить файл подкачки на свой жесткий диск, вы должны создать файл подкачки. Используйте команду DD для создания файла подкачки на жестком диске. Ниже приведена команда DD, используемая для создания файла подкачки (/var/swap1) размером 32 МБ.

```
# dd if=/dev/zero of=/var/swap1 bs=1k count=32768
32768+0 records in
32768+0 records out
33554432 bytes transferred in 3.259 secs (10295928 bytes/sec)
```

После этого вы устанавливаете разрешение на использование файла. Цель состоит в том, чтобы не все пользователи могли использовать файл подкачки.

```
# chmod 0600 /var/swap1
```
Затем вы добавляете файл подкачки в список устройств подкачки ядра с помощью команды swapctl, как в следующем примере.

```
# swapctl -a /var/swap1
```

Теперь вы проверяете, что файл был правильно добавлен на устройство подкачки.

```
# swapctl -l
Device      512-blocks     Used        Avail    Capacity  Priority
/dev/wd0b      4160768         0   4160768         0%         0
/var/swap1         65536         0       65536          0%        0
Total               4226304         0   4226304         0%
```

В OpenBSD все файлы подкачки хранятся в /etc/fstab. Вы можете просмотреть файл подкачки, созданный вами выше, с помощью следующей команды.

```
# cat /etc/fstab
3b48513b4b0fa04e.b none swap sw
3b48513b4b0fa04e.a / ffs rw 1 1
3b48513b4b0fa04e.k /home ffs rw,nodev,nosuid 1 2
3b48513b4b0fa04e.d /tmp ffs rw,nodev,nosuid 1 2
3b48513b4b0fa04e.f /usr ffs rw,nodev 1 2
3b48513b4b0fa04e.g /usr/X11R6 ffs rw,nodev 1 2
3b48513b4b0fa04e.h /usr/local ffs rw,wxallowed,nodev 1 2
3b48513b4b0fa04e.j /usr/obj ffs rw,nodev,nosuid 1 2
3b48513b4b0fa04e.i /usr/src ffs rw,nodev,nosuid 1 2
3b48513b4b0fa04e.e /var ffs rw,nodev,nosuid 1 2
```

### 2.2. Включение Файла Подкачки При Загрузке

В NetBSD и OpenBSD инструмент swapctl можно использовать для активации устройства или файла подкачки во время загрузки. Следующие две команды часто выполняются по умолчанию в NetBSD и OpenBSD startup, чтобы включить все устройства подкачки блочного типа и файлы подкачки, перечисленные в /etc/fstab (с "sw").

С помощью параметра-t команда swapctl может указать тип устройства, которое будет добавлено для определения. Аргумент blk приводит к добавлению всех блочных устройств в /etc / fstab. Аргумент noblk приводит к добавлению всех неблокируемых устройств в /etc/fstab.

Эта опция полезна при первоначальном запуске системы, когда может потребоваться замена до того, как будут доступны все файловые системы, например, при проверке диска в больших файловых системах.

```
# swapctl -A -t blk
# swapctl -A -t noblk
```

### 2.3. Программное обновление

Программное обновление было разработано Грегом Гангером (Greg Ganger) и Йелем Паттом (Yale Patt) и разработано для FreeBSD Кирком Маккьюзиком (Kirk McKusick). Программное обновление частично изменило порядок операций с буферным кэшем без удаления кода синхронизации записи FFS в каталоге. Это приводит к значительному ускорению операций записи.

Механизм мягкого обновления должен быть активирован на этапе установки. При установке с использованием в команде mount вы должны указать, следует ли применять программные обновления к этому разделу. Ниже приведен пример включения программных обновлений для одного раздела sd0a. Откройте файл /etc/fstab, введите сценарий, подобный приведенному ниже.

```
/dev/sd0a/ffsrw,softdep 1 1
```

### 2.4. Удаление файла подкачки

Если созданный вами файл подкачки больше не используется или вы хотите изменить его размер. Существующий файл подкачки вы можете удалить с помощью следующей команды.

```
# swapctl -d /var/swap1
```
