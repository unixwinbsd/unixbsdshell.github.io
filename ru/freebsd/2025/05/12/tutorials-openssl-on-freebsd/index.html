    Учебники OpenSSL на FreeBSD - Установка и настройка SSL сертификатов на FreeBSD

    Одной из программ, требующих особого внимания со стороны пользователей FreeBSD, является набор программ OpenSSL, включенный по умолчанию в систему FreeBSD. OpenSSL предоставляет уровень шифрования поверх уровня обычного подключения; поэтому он может использоваться многими сетевыми приложениями и службами.

    OpenSSL можно использовать для шифрования подключений к почтовым клиентам, интернет-транзакций, таких как кредитные карты, и многого другого. Многие порты, такие как www/apache13-ssl и mail / sylpheed-claws, созданы с использованием OpenSSL.

    OpenSSL это инструментарий с открытым исходным кодом и криптографическая библиотека, реализующая протоколы SSL (Secure Sockets Layer) и TLS (Transport Layer Security).

    OpenSSL это независимый проект, управляемый добровольцами со всего мира. Короче говоря, OpenSSL предоставляет криптографическое устройство для защиты сетевых подключений. Распространенной реализацией SSL является защита веб-страниц с помощью HTTPS (протокол передачи гипертекста, защищенный шифрованием). Шифрование OpenSSL по протоколу HTTP состоит из следующих этапов:

    1. HTTP-клиент (веб-браузер) отправляет HTTPS-запрос на веб-сервер.
    2. В ответ сервер отправляет клиенту SSL-сертификат, содержащий открытый ключ, доменное имя и выданный центр сертификации SSL.
    3. Клиенты отправляют сообщения, зашифрованные с использованием открытого SSL-ключа сервера.
    4. Сервер расшифровывает это сообщение с помощью своего закрытого SSL-ключа.
    5. В конечном итоге сервер отправляет клиенту расшифрованное сообщение.
    6. Если клиент получает правильное сообщение, обе стороны могут начать безопасный обмен информацией при условии, что клиент доверяет центру выдачи сертификатов.

    OpenSSL предоставляет необходимые инструменты для создания запросов на подписание сертификатов, закрытых серверных ключей и самозаверяющих сертификатов. В сочетании с признанным центром сертификации можно создать сертификат доверенного сервера для использования с протоколами TCP, такими как HTTP, SMTP, IMAP и т.д.

    OpenSSL основан на библиотеке SSLeay, первоначально разработанной Эриком А. Янгом и Тимом Дж. Хадсоном. SSLeay - это реализация протокола Secure Socket Layer от Netscape с открытым исходным кодом, который использовался в браузерах Netscape Secure Server и Navigator в середине 1990-х годов.

    Версия OpenSSL, входящая в состав FreeBSD, поддерживает протоколы сетевой безопасности Secure Sockets Layer v2/v3 (SSLv2/SSLv3) и Transport Layer Security v1 (TLSv1) и может использоваться в качестве основной криптографической библиотеки.

    1. Обновите OpenSSL

    Чаще всего OpenSSL используется для создания сертификатов, используемых пакетами программного обеспечения. Этот сертификат подтверждает, что данные компании или физического лица указаны правильно и не изменены. Если соответствующий сертификат не был проверен одним из нескольких центров сертификации (Cas), обычно выводится предупреждение.

    Центр сертификации - это компания, такая как VeriSign, которая подписывает сертификат для проверки личности физического лица или компании. Эта процедура не является бесплатной и не является абсолютно необходимой для использования сертификата; однако она может убедить некоторых очень осторожных пользователей.

    В последних версиях FreeBSD вам не нужно устанавливать OpenSSL, поскольку OpenSSL автоматически устанавливается в системах FreeBSD. Несмотря на это, иногда встроенная в систему версия не является последней. Нет ничего плохого в том, что вы обновите его, переустановив OpenSSL. Процесс установки OpenSSL очень прост. Вот как установить OpenSSL в систему FreeBSD.

    root@ns4:~ # cd /usr/ports/security/openssl
root@ns4:/usr/ports/security/openssl # make DISABLE_VULNERABILITIES=yes install clean
root@ns4:/usr/ports/security/openssl # echo "DEFAULT_VERSIONS+=ssl=openssl" >> /etc/make.conf

    В процессе установки появится меню выбора зависимостей. Вы можете включить или отключить показанные опции, как показано на следующем рисунке.

    После этого приступайте к процессу настройки. Перед настройкой OpenSSL создайте резервную копию или скопируйте файлы openssl.cnf, чтобы в случае возникновения ошибки во время настройки исходный файл оставался на месте.

    root@ns4:~ # cd /usr/local/openssl
root@ns4:/usr/local/openssl # cp openssl.cnf openssl.cnf.backup

    Теперь вы можете напрямую изменить файл openssl.cnf, но перед этим сначала проверьте версию OpenSSL, которую вы используете.

    root@ns4:~ # openssl version
OpenSSL 3.0.16 11 Feb 2025 (Library: OpenSSL 3.0.16 11 Feb 2025)

    Поскольку FreeBSD установила OpenSSL в качестве программы по умолчанию, перед установкой OpenSSL была произведена системная настройка OpenSSL. Мы просто следуем конфигурации системы по умолчанию, удалив файл /usr/local/openssl/openssl.cnf и создав символическую ссылку из файла /etc/ssl / openssl.cnf.

    root@ns4:~ # rm -f /usr/local/openssl/openssl.cnf
root@ns4:~ # ln -s /etc/ssl/openssl.cnf /usr/local/openssl/openssl.cnf

    С помощью приведенной выше команды основной файл конфигурации OpenSSL находится по адресу /etc/ssl / openssl.cnf. Любые изменения в файле /etc/ssl/openssl.cnf повлияют на конфигурацию всей системы.

    2. Создайте сертификат CA

    В этом разделе SSL-сертификат будет создан с помощью скрипта CA.pl (Perl-скрипт), установленного через порт OpenSSL. Вы можете создать официальный сертификат для подписи вашего SSL-сертификата, отправив запрос. Центру сертификации требуется файл запроса сертификата для создания действительного SSL-сертификата для вашего сервера. Мы будем использовать скрипт CA.pl, который входит в состав OpenSSL, для выполнения запросов сертификатов. Следующий скрипт скопирует файл CA.pl перейдите в папку /usr/local/openssl/certs.

    root@ns4:~ # cd /usr/local/openssl
root@ns4:/usr/local/openssl # cp misc/CA.pl certs

    Выполните приведенную ниже команду, чтобы создать запрос на сертификат.

    root@ns4:~ # cd /usr/local/openssl/certs
root@ns4:/usr/local/openssl/certs # ./CA.pl -newreq
Use of uninitialized value $1 in concatenation (.) or string at ./CA.pl line 145.
====
openssl req  -new  -keyout newkey.pem -out newreq.pem -days 365
Ignoring -days without -x509; not generating a certificate

    При создании вышеуказанного сертификата вы должны ввести пароль, используйте пароль, который легко запомнить. Обратите внимание на “Общее имя” (т.е. полное доменное имя сервера или ваше имя). В этом разделе мы должны ввести имя хоста и домен сервера FreeBSD. В этой статье “ns4” - это имя хоста, а “unixexplore.com” - доменное имя используемого в данный момент сервера FreeBSD.

    Запустите файл скрипта. CA.pl приведенные выше действия сгенерируют файлы “newkey.pem” и “newreq.pem”.

    root@ns4:/usr/local/openssl/certs # ls -l
total 14
-rwxr-xr-x  1 root wheel 8064 May 13 07:15 CA.pl
-rw-------  1 root wheel    0 May 13 07:27 newkey.pem
-rw-r--r--  1 root wheel  956 May 13 07:26 newreq.pem

    Оба файла содержат зашифрованный SSL-сертификат для частного сервера. Для упрощения идентификации скопируйте файл в ns4.unixexplore.com-encrypted-key.pem, используя следующую команду.

    root@ns4:/usr/local/openssl/certs # cp newkey.pem ns4.unixexplore.com-encrypted-key.pem
root@ns4:/usr/local/openssl/certs # cp newreq.pem ns4.unixexplore.com-req.pem

    Файл ns4.unixexplore.com-encrypted-key.pem зашифрован с помощью пароля, который вы ввели ранее. Вам следует всегда помнить этот пароль. Вы должны ввести его при запуске приложения SSL.

    Если этот файл будет использоваться на сервере без присмотра, то, возможно, будет хорошей идеей зашифровать его, чтобы демон мог запускать его без участия пользователя. Чтобы удалить шифрование и сделать файл нешифрованным, доступным только для чтения пользователем root, воспользуйтесь следующим скриптом.

    root@ns4:/usr/local/openssl/certs # openssl rsa -in ns4.unixexplore.com-encrypted-key.pem -out ns4.unixexplore.com-unencrypted-key.pem
Enter pass phrase for ns4.unixexplore.com-encrypted-key.pem:
writing RSA key
root@ns4:/usr/local/openssl/certs # chmod 400 ns4.unixexplore.com-unencrypted-key.pem

    3. Создайте самозаверяющий SSL-сертификат

    Одним из способов создания собственного сертификата является использование файла CA.pl. Пожалуйста, обратите внимание, что при создании собственного сертификата в клиенте приложения (веб-браузере, почтовом клиенте и т.д.) появится диалоговое окно “Ненадежный сертификат”. Чтобы избежать этого, вы можете установить файл сертификата вашего сервера в клиентской системе. Вот скрипт для создания вашего собственного SSL-сертификата сроком действия 3 года.

    root@ns4:~ # cd /usr/local/openssl
root@ns4:/usr/local/openssl # cp misc/CA.pl certs
root@ns4:/usr/local/openssl # sed -I .old 's/365/1095/' /etc/ssl/openssl.cnf

    Выполните следующую команду, чтобы создать свой собственный центр сертификации SSL.

    root@ns4:/usr/local/openssl # cd /usr/local/openssl/certs
root@ns4:/usr/local/openssl/certs # ./CA.pl -newca

    Чтобы создать вышеуказанный запрос на сертификат, используйте следующую команду.

    root@ns4:/usr/local/openssl/certs # ./CA.pl -newreq
Use of uninitialized value $1 in concatenation (.) or string at ./CA.pl line 145.
====
openssl req  -new  -keyout newkey.pem -out newreq.pem -days 365
Ignoring -days without -x509; not generating a certificate

    Используя команду copy (cp), скопируйте все сертификаты в каталог /usr/local/openssl/certs, используя для этого следующий скрипт.

    root@ns4:/usr/local/openssl/certs # cp newreq.pem ns4.unixexplore.com-cert.pem
root@ns4:/usr/local/openssl/certs # cp newkey.pem ns4.unixexplore.com-encrypted-key.pem
root@ns4:/usr/local/openssl/certs # cp demoCA/cacert.pem ./unixexplore.com-CAcert.pem
root@ns4:/usr/local/openssl/certs # cp demoCA/private/cakey.pem ./unixexplore.com-encrypted-CAkey.pem

    Чтобы удалить шифрование и создать незашифрованный файл, который может быть прочитан только пользователем root, используйте следующую команду.

    root@ns4:/usr/local/openssl/certs # openssl rsa -in ns4.unixexplore.com-encrypted-key.pem -out ns4.unixexplore.com-unencrypted-key.pem
Enter pass phrase for ns4.unixexplore.com-encrypted-key.pem:
writing RSA key
root@ns4:/usr/local/openssl/certs # chmod 400 ns4.unixexplore.com-unencrypted-key.pem

    Теперь мы экспортируем сертификат CA или корневой сертификат, чтобы его можно было установить в систему, которая будет использовать ваш SSL-сертификат. Это необходимо для предотвращения появления предупреждающего сообщения о ненадежном корневом SSL-сертификате. Это сообщение предупреждает конечного пользователя о потенциальной проблеме с SSL-сертификатом.

    Если эти ненужные предупреждения не будут удалены, будет сложнее обнаружить фактический перехват SSL-сеанса. Большинство клиентских систем (Windows и Mac OS X) распознают файлы SSL-сертификатов, закодированные в двоичном формате DER (Distincted Encoding Rules). Чтобы преобразовать текстовый сертификат Pem (Privacy Enhanced Mail) в формат DER, введите следующую команду.

    root@ns4:/usr/local/openssl/certs # openssl x509 -in unixexplore.com-CAcert.pem -inform PEM -out unixexplore.com-CAcert.cer -outform DER

    Вы можете отправить сертификаты с кодом DER по электронной почте с помощью следующей команды.

    root@ns4:/usr/local/openssl/certs # uuencode unixexplore.com-CAcert.cer unixexplore.com-CAcert.cer | mail -s "Subject" datainchi@gmail.com

    CAcert - это центр сертификации, который выдает бесплатные SSL-сертификаты. Если вы новичок в использовании SSL-сертификатов, ознакомьтесь с рассылкой LinkedIn UNIX BSD Explore, посвященной процессу подписания SSL-сертификатов. Вы можете создать запрос на подписание сертификата, как описано в этом руководстве, и подписать его с помощью CAcert.

    Сертификаты, подписанные CAcert, имеют ограниченную поддержку. Большинство веб-браузеров не включают свои корневые сертификаты в базы данных доверенных центров сертификации. Однако корневые сертификаты CAcert включены во FreeBSD и некоторые дистрибутивы Linux.
