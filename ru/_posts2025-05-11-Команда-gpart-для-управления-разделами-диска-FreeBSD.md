### г. Изменение атрибутов раздела - gpart modify
Команда gpart modify может изменить только тип раздела и метку раздела. Чтобы изменить тип и метку раздела, необходимо указать его с помощью параметра -i (индекс).

Чтобы использовать каждую опцию изменения gpart, вы можете ознакомиться с ней в таблице ниже:

| **Option**     | **Description**          | 
| -------------  | ---------------------------------------------------------------------------------------  | 
| -f flags   | Additional operational flags. See the section entitled  "OPERATIONAL FLAGS" below for a discussion about its use. 						  | 
| -i index       | Specifies the index of the partition to be modified.      | 
| -l label       | Change the partition label to label.     | 
| -t type       | Change the partition type to type.  | 

В таблице выше мы видим, что команда gpart modify может работать только с опциями -l и -t с учетом опции -i.

#### г.1. Изменение метки раздела
В примере ниже мы покажем вам, как изменить метку жесткого диска.
<br/>
```
root@ns4:~ # gpart show -l ada0
=>       40  488397088  ada0  GPT  (233G)
         40       1024     1  gptboot0  (512K)
       1064        984        - free -  (492K)
       2048    4194304     2  swap0  (2.0G)
    4196352  484200448     3  zfs0  (231G)
  488396800        328        - free -  (164K)
```
Обратите внимание на сообщения, отображаемые командой gpart show. В описании выше поясняется, что имя метки жесткого диска — «zfs0» с индексом 3. Мы изменим имя метки на «hard-disk-1» с помощью команды gpart modify, как в примере ниже.

```
root@ns4:~ # gpart modify -i 3 -l hard-disk-1 ada0
ada0p3 modified
```
После изменения имени метки на hard-disk-1 выполните команду gpart show, чтобы увидеть, изменилось ли имя метки или нет.
<br/>
```
root@ns4:~ # gpart show -l ada0
=>       40  488397088  ada0  GPT  (233G)
         40       1024     1  gptboot0  (512K)
       1064        984        - free -  (492K)
       2048    4194304     2  swap0  (2.0G)
    4196352  484200448     3  hard-disk-1  (231G)
  488396800        328        - free -  (164K)
```
В приведенной выше команде мы видим, что имя метки «zfs0» было изменено на «hard-disk-1».


### г.2. Изменение типа раздела
С помощью gpart modify вы также можете изменить тип раздела жесткого диска. Вы можете изменить тип существующих разделов, см. разделы Flashdisk ниже.

```
root@ns4:~ # gpart show da0
=>      40  30834608  da0  GPT  (15G)
        40      2008       - free -  (1.0M)
      2048  14680064    1  freebsd-ufs  (7.0G)
  14682112   2097152    2  freebsd-ufs  (1.0G)
  16779264   4194304    3  freebsd-ufs  (2.0G)
  20973568   9861080       - free -  (4.7G)
```

Существует три типа разделов, а именно freebsd-ufs с индексами 1, 2 и 3. Поскольку вы хотите создать файл подкачки, мы изменим один из разделов freebsd-ufs на раздел подкачки, например, мы изменим индекс 2 на swap. Для этого просто выполните команду ниже.

```
root@ns4:~ # gpart modify -i 2 -t freebsd-swap da0
da0p2 modified
```

Теперь давайте посмотрим на изменения, произошедшие после выполнения команды выше с помощью gpart show.

```
root@ns4:~ # gpart show da0
=>      40  30834608  da0  GPT  (15G)
        40      2008       - free -  (1.0M)
      2048  14680064    1  freebsd-ufs  (7.0G)
  14682112   2097152    2  freebsd-swap  (1.0G)
  16779264   4194304    3  freebsd-ufs  (2.0G)
  20973568   9861080       - free -  (4.7G)
```
### ч. Изменить размер раздела - gpart resize
Каждый раздел имеет предопределенный размер, но вы можете изменить размер раздела по своему желанию. Для изменения размера существующего раздела на диске можно использовать команду gpart resize.

```
root@ns4:~ # gpart show da0
=>      40  30834608  da0  GPT  (15G)
        40       512    1  freebsd-boot  (256K)
       552   2097152    2  freebsd-swap  (1.0G)
   2097704  12582912    3  freebsd-ufs  (6.0G)
  14680616  16154032       - free -  (7.7G)
```

В приведенной выше команде в конце диска имеется свободное место размером около 7,7 ГБ. Поскольку размер разделов можно изменять только до смежных участков пространства, чтобы изменить размер второго раздела (т. е. корневого раздела), нам нужно удалить третий раздел (что безопасно, поскольку раздел подкачки содержит только временные данные), изменить размер второго раздела, а затем заново создать раздел подкачки.

Если вы активировали файл подкачки с помощью команды «swapon /dev/da0», сначала отключите файл подкачки в da0.

```
root@ns4:~ # swapoff /dev/da0
```
Следующий шаг — удаление раздела подкачки. Поскольку файл подкачки находится в индексе №. 2, вы удаляете файл подкачки с помощью команды gpart delete.

```
root@ns4:~ # gpart delete -i 2 /dev/da0
```
Посмотрите изменения, произошедшие после команды удаления раздела подкачки.

```
root@ns4:~ # gpart show /dev/da0
=>      40  30834608  da0  GPT  (15G)
        40       512    1  freebsd-boot  (256K)
       552   2097152       - free -  (1.0G)
   2097704  12582912    3  freebsd-ufs  (6.0G)
  14680616  16154032       - free -  (7.7G)
```

Затем меняем размер раздела freebsd-ufs на 12G. Обратите внимание на порядковый номер раздела freebsd-ufs. Чтобы изменить размер раздела freebsd-ufs с 6,0 ГБ до 12 ГБ, выполните команду ниже.

```
root@ns4:~ # gpart resize -i 3 -s 12G -a 4k da0
da0p3 resized
```
Как обычно, просмотрите результаты с помощью команды gpart show.

```
root@ns4:~ # gpart show da0
=>      40  30834608  da0  GPT  (15G)
        40       512    1  freebsd-boot  (256K)
       552   2097152       - free -  (1.0G)
   2097704  25165824    3  freebsd-ufs  (12G)
  27263528   3571120       - free -  (1.7G)
```
После выполнения команды выше осталось еще 1,7 Гб свободного места. Оставшийся объем мы используем для хранения файла подкачки. Чтобы создать раздел подкачки, вы можете выполнить следующую команду.

```
root@ns4:~ # gpart add -t freebsd-swap -a 4k da0
da0p2 added
```
### я. Установка загрузочного кода на раздел - gpart bootcode
Чтобы установить загрузочный код в метаданные схемы разделов на диске, используйте команду gpart bootcode. Например, чтобы установить загрузочный код GPT на da0, используйте:

```
root@ns4:~ # gpart bootcode -b /boot/pmbr -p /boot/gptboot -i 1 da0
partcode written to da0p1
bootcode written to da0
```

Приведенная выше команда установит загрузочный код GPT на первый раздел -i (индекс 1) на диске da0. При этом параметр -b указывает загрузочный код PMBR (Protective MBR), а параметр -p указывает загрузочный код GPT.

Если вы хотите использовать загрузочный код ZFS, замените gptboot на gptzfsboot.

```
root@ns4:~ # gpart bootcode -b /boot/pmbr -p /boot/gptzfsboot -i 1 da0
partcode written to da0p1
bootcode written to da0
```
### дж. Создать резервный раздел - gpart backup
Создание резервного раздела очень важно для предотвращения сбоя системы. С помощью резервного копирования разделов мы можем легко восстановить поврежденные или утерянные разделы.

Резервный раздел можно создать с помощью команды gpart backup. Эта команда отображает информацию о таблице разделов в специальном текстовом формате. Вы можете перенаправить эту информацию в файл или передать ее команде «gpart restore» для восстановления раздела.

Вот простая команда резервного копирования gpart.

```
root@ns4:~ # gpart backup ada0
GPT 128
1   freebsd-boot        40      1024 gptboot0
2   freebsd-swap      2048   4194304 swap0
3    freebsd-zfs   4196352 484200448 zfs0
```

Чтобы команда резервного копирования gpart могла работать оптимально во время восстановления. Лучше создать специальный файл для резервного раздела, как в следующем примере.

```
root@ns4:~ # gpart backup da0 > /mnt/da0.backup
```
### к. Восстановление раздела - gpart restore
Если вы случайно уничтожили таблицу разделов схемы GPT, у вас все равно есть несколько вариантов ее восстановления. Обратите внимание, я сделал резервную копию раздела с помощью команды gpart backup, если нет, то вам предстоит сделать еще кое-что.

Итак, если вы ранее создали резервную копию таблицы разделов с помощью команды gpart backup, то вам повезло, и, скорее всего, вам понадобится всего несколько команд оболочки, чтобы вернуть вашу систему в нормальное состояние.

Мы приведем пример запуска команды gpart restore для восстановления удаленного раздела. Выполняйте каждую продемонстрированную нами команду.

#### к.1. Удалить раздел
```
root@ns4:~ # gpart destroy -F /dev/da0
```

#### к.2. Создайте таблицу схем и типов разделов.
```
root@ns4:~ # gpart create -s gpt da0
root@ns4:~ # gpart add -s 64k -t freebsd-boot da0
root@ns4:~ # gpart add -s 6G -t freebsd-ufs da0
root@ns4:~ # gpart add -t freebsd-swap da0
root@ns4:~ # gpart bootcode -b /boot/pmbr -p /boot/gptboot -i 1 da0
```
Теперь вы можете увидеть содержимое разделов на диске da0.
```
root@ns4:~ # gpart show da0
=>      40  30834608  da0  GPT  (15G)
        40       128    1  freebsd-boot  (64K)
       168  12582912    2  freebsd-ufs  (6.0G)
  12583080  18251568    3  freebsd-swap  (8.7G)
```

Приведенные выше команды полностью соответствуют настройкам FreeBSD по умолчанию во время процесса установки.

1. Первый раздел должен называться freebsd-boot, а его размер, скорее всего, составит 64 КБ.
2. Второй раздел — корневой раздел (freebsd-ufs).
3. Третий раздел — это раздел подкачки (freebsd-swap).
4. Остальные разделы зависят от вашей конфигурации.

#### к.3. Создать резервный раздел
```
root@ns4:~ # gpart backup da0 > /mnt/da0.backup
```

#### к.4. Удалить раздел

```
root@ns4:~ # gpart destroy -F /dev/da0
```

#### к.5. Восстановить раздел

```
root@ns4:~ # gpart restore -F da0 < /mnt/da0.backup
```

В команде восстановления опция -F сначала полностью сотрет все существующие данные из таблицы разделов, а затем восстановит таблицу из файлов.

Подробно рассмотрены основные команды gpart в man gpart. Вы просто запоминаете каждую выполняемую команду. Если у вас возникли трудности с изучением команды gpart, вы можете оставить комментарий или напрямую посетить официальный сайт gpart.
